# 项目管理工具详解

## 一、规划工具

### 1. WBS工作分解结构
**定义**：将项目分解为可管理的小块

**步骤**：
1. 识别项目主要交付物
2. 分解为子交付物
3. 分解为工作包
4. 分配责任

**原则**：
- 100%原则：包含所有工作
- 相互独立：不重叠
- 层级清晰：3-6层

### 2. 关键路径法(CPM)
**定义**：项目中最长的任务序列

**计算**：
- ES(最早开始) = max(所有前置任务EF)
- EF(最早完成) = ES + 工期
- LS(最晚开始) = LS - 工期
- LS(最晚完成) = min(所有后续任务LS)
- 浮动时间 = LS - ES

**关键路径**：浮动时间为0的任务序列

### 3. 资源平衡/平滑
- **资源平衡**：调整任务解决资源冲突，可能延长工期
- **资源平滑**：调整任务在浮动时间内解决资源冲突，不延长工期

## 三、成本控制工具

### 1. EVM挣值管理
**核心指标**：
```
PV = Budgeted Cost of Work Scheduled
EV = Budgeted Cost of Work Performed  
AC = Actual Cost of Work Performed

CV = EV - AC (成本偏差)
SV = EV - PV (进度偏差)

CPI = EV/AC (成本绩效)
SPI = EV/PV (进度绩效)

EAC = BAC/CPI (完工估算)
ETC = EAC - AC (完工尚需)
VAC = BAC - EAC (完工偏差)
```

**绩效评估**：
| CPI | SPI | 含义 | 措施 |
|-----|-----|------|------|
| >1 | >1 | 超预期 | 保持 |
| >1 | <1 | 成本好进度差 | 加速 |
| <1 | >1 | 进度好成本超 | 控成本 |
| <1 | <1 | 差 | 补救 |

### 2. 完工估算方法
- EAC = AC + BAC-EV (非典型)
- EAC = AC + ETC (典型)
- EAC = BAC/CPI (最可能)

## 四、质量管理工具

### 1. 质量工具
- 检查表
- 帕累托图
- 鱼骨图
- 控制图
- 直方图
- 散点图
- 流图

### 2. FMEA失效模式
**步骤**：
1. 识别潜在失效模式
2. 评估严重度(S)
3. 评估发生率(O)
4. 评估检测度(D)
5. 计算RPN = S × O × D
6. 制定改进措施

**RPN优先级**：数值越高越优先改进

### 1. 风险识别
- 头脑风暴
- 德尔菲法
- 核对表

### 2. 风险评估
**概率影响矩阵**：
|  | 低影响 | 中影响 | 高影响 |
|--|--------|--------|--------|
|高概率| 中 | 高 | 极高 |
|中概率| 低 | 中 | 高 |
|低概率| 低 | 低 | 中 |

### 3. 风险应对
| 策略 | 说明 | 例子 |
|------|------|------|
| 规避 | 改变计划 | 取消高风险范围 |
| 转移 | 转嫁风险 | 买保险、外包 |
| 减轻 | 降低概率/影响 | 测试、备份 |
| 接受 | 承认并准备 | 应急计划 |

## 六、沟通管理工具

### 1. 沟通模型
```
发送者 → 编码 → 媒介 → 解码 → 接收者
         ↓
      反馈
```

### 2. 沟通方式选择
| 方式 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| 邮件 | 留档、可复制 | 慢 | 详细信息 |
| 即时消息 | 快 | 无记录 | 紧急 |
| 会议 | 讨论充分 | 费时 | 决策 |
| 面对面 | 充分沟通 | 成本高 | 重要 |

### 3. 干系人分析矩阵
```
            高利益
              ↑
              │  重点管理
   保持满意 ←─┼─→ 随时告知
              │
   监督 ←─────┼─────→ 令其满意
              │
            低利益
              ↓
          低影响 → 高影响
```
