# 项目管理知识体系

## 一、项目管理基础

### 1. 项目定义
项目是为创造独特产品、服务或成果而进行的临时性工作。

### 2. 项目特征
- **临时性**：有明确的开始和结束
- **独特性**：每个项目都是独特的
- **渐进明细**：逐步清晰

### 3. 项目管理五大过程组
```
启动 → 规划 → 执行 → 监控 → 收尾
```

---

## 二、十大知识领域

### 1. 整合管理
**核心**：协调所有其他知识领域

**过程**：
- 制定项目章程
- 制定项目管理计划
- 指导与管理项目工作
- 管理项目知识
- 监控项目工作
- 实施整体变更控制
- 结束项目或阶段

### 2. 范围管理
**核心**：确保项目做且仅做所需工作

**工具**：
- WBS工作分解结构
- 范围说明
- 范围基准

**输出**：
- 项目范围说明书
- WBS
- WBS词典

### 3. 进度管理
**核心**：按时完成

**工具**：
- 甘特图
- 关键路径法(CPM)
- 关键链法(CCM)
- 资源平衡

**输出**：
- 项目进度计划
- 进度基准

### 4. 成本管理
**核心**：在预算内完成

**工具**：
- 成本估算
- 预算制定
- 成本控制
- EVM挣值管理

**公式**：
```
PV = 计划值
EV = 挣值  
AC = 实际成本
CV = EV - AC (成本偏差)
SV = EV - PV (进度偏差)
CPI = EV/AC (成本绩效)
SPI = EV/PV (进度绩效)
```

### 5. 质量管理
**核心**：满足需求

**工具**：
- 质量规划
- 质量保证
- 质量控制
- PDCA循环

### 6. 资源管理
**核心**：有效管理资源

**工具**：
- 资源分解结构(RBS)
- 责任分配矩阵(RAM)
- 团队建设
- 资源平滑

### 7. 沟通管理
**核心**：及时准确传递信息

**沟通方式**：
- 正式/非正式
- 口头/书面
- 对内/对外

**频率**：
- 日报/周报
- 周会/月会
- 项目例会

### 8. 风险管理
**核心**：识别分析应对风险

**风险流程**：
1. 风险管理计划
2. 识别风险
3. 定性分析
4. 定量分析
5. 规划应对
6. 实施应对
7. 监督风险

**应对策略**：
- 规避
- 转移
- 减轻
- 接受

### 9. 采购管理
**核心**：外部采购

**合同类型**：
- 固定总价(FP)
- 成本加固定费用(CPFF)
- 时间材料(T&M)

### 10. 干系人管理
**核心**：满足干系人需求

**干系人分类**：
- 内部/外部
- 高/低权力
- 高/低利益
