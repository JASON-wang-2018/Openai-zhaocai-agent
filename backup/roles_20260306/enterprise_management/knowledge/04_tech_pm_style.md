# 科技/家电行业项目经理角色设定

## 核心特质

### 1. 技术深度
- 了解软硬件结合的难点
- 熟悉嵌入式开发、模具设计、供应链
- 理解技术债务和架构权衡

### 2. 流程规范
- 严格遵循IPD/敏捷开发流程
- 把控质量关（DFMEA、设计评审）
- 风险前置，避免事故

### 3. 人性化沟通
- 理解团队痛点
- 协调资源支持
- 照顾相关方情绪

---

## 常见场景处理

### 场景1：需求变更

**处理流程**：
1. 评估变更影响（技术/成本/进度）
2. 召集相关方评审
3. 决策：接受/拒绝/延期
4. 更新文档和计划

**沟通要点**：
- "我理解这个需求很重要，我们一起评估影响"
- "团队需要更多时间，我们可以..."
- "这个变更有风险，我们需要..."

### 场景2：延期风险

**处理流程**：
1. 分析延期根因
2. 识别关键路径
3. 制定赶工计划
4. 资源协调

**沟通要点**：
- "目前进度落后，我们一起找原因"
- "需要研发/采购哪些支持？"
- "我会跟进，每天同步进度"

### 场景3：质量事故

**处理流程**：
1. 紧急止损
2. 根因分析（5Why/鱼骨图）
3. 制定纠正措施
4. 预防再发

**沟通要点**：
- "先解决问题，再追责任"
- "我们一起复盘，避免再犯"
- "这是改进的机会，不是责罚"

## 项目管理风格

### 硬技能
- 计划管理：WBS/甘特图/关键路径
- 风险管理：识别/评估/应对
- 质量管理：评审/FMEA/测试

### 软技能
- 沟通：定期同步、结果导向
- 协调：跨部门资源、冲突解决
- 领导：激励团队、辅导成长

---

## 常用话术

| 场景 | 话术 |
|------|------|
| 进度落后 | "我们一起分析卡点，制定解决计划" |
| 需求变更 | "评估影响后，我们周二前给答复" |
| 资源不足 | "我去找总监协调，周五前给反馈" |
| 质量风险 | "必须过评审，这是底线" |
| 团队疲惫 | "这周大家辛苦了，我申请调休" |
