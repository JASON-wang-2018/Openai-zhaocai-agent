# 项目管理知识库

## 一、项目管理基础

### 1. 项目定义
项目是为创造独特产品、服务或成果而进行的临时性工作。

### 2. 项目特征
- 临时性（有时间限制）
- 独特性（创新）
- 渐进明细（逐步清晰）

### 3. 项目生命周期
```
启动 → 规划 → 执行 → 监控 → 收尾
```

### 3. 执行过程组

### 4. 监控过程组
- 监控项目工作

### 5. 收尾过程组

- 合同收尾

## 三、项目管理知识领域

### 1. 整合管理
**核心**：协调所有其他知识领域

**过程**：
- 制定项目章程
- 制定项目管理计划
- 指导与管理项目工作
- 管理项目知识
- 监控项目工作
- 实施整体变更控制
- 结束项目或阶段

### 2. 范围管理
**核心**：确保项目做且仅做所需工作

**工具**：
- WBS工作分解结构
- 范围说明
- 范围基准

### 3. 进度管理
**核心**：按时完成项目

**工具**：
- 甘特图
- 关键路径法(CPM)
- 关键链法(CCM)

### 4. 成本管理
**核心**：在批准的预算内完成项目

**工具**：
- 成本估算
- 预算制定
- 成本控制
- EVM挣值管理

### 5. 质量管理
**核心**：确保项目满足需求

**工具**：
- 质量规划
- 质量保证
- 质量控制

### 6. 资源管理
**核心**：有效管理项目资源

**工具**：
- 资源分解结构(RBS)
- 责任分配矩阵(RAM)
- 团队建设

### 7. 沟通管理
**核心**：确保项目信息及时准确

**沟通方式**：
- 正式/非正式
- 口头/书面
- 对内/对外

### 8. 风险管理
**核心**：识别分析应对风险

**风险流程**：
1. 风险管理计划
2. 识别风险
3. 定性风险分析
4. 定量风险分析
5. 规划风险应对
6. 实施风险应对
7. 监督风险

### 9. 采购管理
**核心**：从项目团队外采购产品服务

**合同类型**：
- 固定总价合同(FP)
- 成本加固定费用(CPFF)
- 时间材料合同(T&M)

### 10. 干系人管理
**核心**：满足干系人需求

**干系人分析**：
- 识别干系人
- 分析期望
- 制定策略

## 四、项目管理工具

### 1. WBS工作分解结构
```
项目
├── 子项目1
│   ├── 工作包1.1
│   └── 工作包1.2
├── 子项目2
│   └── 工作包2.1
└── 子项目3
```

### 2. 责任分配矩阵(RAM)

| 任务 | 张三 | 李四 | 王五 |
|------|------|------|------|
| 设计 | R | C | I |
| 开发 | C | R | I |
| 测试 | I | C | R |

R=责任 C=执行 I=知情

### 3. 甘特图

| 任务 | 周1 | 周2 | 周3 | 周4 |
|------|-----|-----|-----|-----|
| 设计 | ████ |    |    |    |
| 开发 |    | ██████ |    |    |
| 测试 |    |    | ████ |    |
| 上线 |    |    |    | ██ |

### 4. 关键路径法(CPM)
识别项目中最长的任务序列

### 5. 挣值管理(EVM)

| 指标 | 公式 | 含义 |
|------|------|------|
| PV | 计划值 | 计划应完成 |
| EV | 挣值 | 实际已完成 |
| AC | 实际成本 | 实际已花费 |
| CV | 成本差异 | EV-AC |
| SV | 进度差异 | EV-PV |
| CPI | 成本绩效 | EV/AC |
| SPI | 进度绩效 | EV/PV |
